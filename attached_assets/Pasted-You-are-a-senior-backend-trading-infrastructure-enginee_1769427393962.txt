You are a senior backend + trading infrastructure engineer.

Objective:
Build a VPS-hosted execution gateway that connects my SaaS app (FastAPI backend on Render, Next.js frontend) with MetaTrader 5 (MT5) trading accounts.

This gateway must behave like a self-hosted MetaApi alternative.

IMPORTANT CONSTRAINTS (DO NOT VIOLATE):
- MT5 cannot run on Render
- MT5 will run only on a Windows VPS
- One MT5 terminal = one trading account (NO multi-account per terminal)
- Users must NEVER manage VPS or MT5 themselves
- My SaaS app controls everything remotely
- No third-party paid MT5 APIs (MetaApi, etc.)

--------------------------------------------------
SYSTEM OVERVIEW
--------------------------------------------------

Architecture:

[ SaaS App on Render ]
- FastAPI (AI logic, strategy, auth, billing)
- Next.js frontend (UI)

        |
        | HTTPS (secure, API key based)
        v

[ VPS – Execution Layer ]
- Central Gateway App (this project)
- One or more MT5 terminals
- One bridge per MT5 terminal

--------------------------------------------------
WHAT THIS VPS APP MUST DO
--------------------------------------------------

This VPS app is NOT a trading strategy app.
It is ONLY an execution + data gateway.

Core responsibilities:

1. Accept authenticated HTTPS requests from Render backend
2. Manage MT5 terminals lifecycle:
   - Create MT5 instance per user account
   - Login to broker using credentials sent from SaaS
   - Start / stop / restart MT5 safely
3. Bridge communication between MT5 and SaaS:
   - Fetch account info (balance, equity, margin)
   - Fetch open positions and orders
   - Fetch market data (ticks / candles)
   - Place, modify, and close orders
4. Route requests correctly:
   - User → Account → MT5 instance
5. Handle failures:
   - MT5 freeze or crash → auto restart
   - Broker disconnect → retry and report status
6. Expose a clean REST + optional WebSocket API
   - SaaS must NOT know internal MT5 details

MT5 Side:
- One MT5 terminal per user account

  a) Python MetaTrader5 bridge (1 process per terminal)
  OR

Gateway App:

- Stateless API design
- Config-driven account provisioning

Security:

- No direct public MT5 access
- Encrypted storage of MT5 credentials
- IP allowlist (Render IPs only)
